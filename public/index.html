What the browser wants

Browsers want script tags!

<script src="myawesomecode.js"></script>

And style tags!

<link rel="stylesheet" href="awesomestyles.css">

But they don't want too many...

While your browser can (and does) download static assets concurrently...
The overhead of initiating lots of HTTP requests can result in very poor performance for your website.

window

In the browser, window is basically a wild global scope party.

The browser tempts you to put your code on window (the cover fee is your soul).

In Summary

  • We have to communicate with browsers using script tags.
  • In most cases, fewer script tags === faster-loading site
  • Hell is real, its name is window
What developers want
Developers want to split their code into modules.
Developers want explicit dependency management.
Developers want to use ECMAScript 2015 (ES6) (and beyond).
Developers want to make websites that load quickly.
Developers want to write JavaScript without entering a manic depressive state.
Obviously, there exists a disparity between what developers want and what they have to deliver to the browser.
What developers do

We cry

Then we engineer a better way. (So more crying.)
We use tools, like Babel, which convert our ES6 code into browser-compatible, vanilla JavaScript. This is called transpilation
Instead of putting 678765434567 script tags on our pages, we harness tools to combine them all into one script which we then serve in production. This is called concatenation
We minify our code in order to serve the fewest bytes possible to clients. This is called minification
(And we do a whole host of things along the way, like Sass, cache-busting, more concatenation, more minification, source maps, auto-prefixing, ...)
Finally, we use a task runner, such as Gulp or Grunt, to wire everything together. We step back to behold our glorious build system...
And we see a thousand-line Gruntfile...
And a package.json with dozens of dependencies just for running the build task...
We've achieved what we wanted...

...but shouldn't there be a better way?
